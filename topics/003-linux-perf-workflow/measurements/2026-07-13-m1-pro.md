# Apple M1 Pro elapsed-time baseline — 2026-07-13

- CPU: Apple M1 Pro
- Operating system: Darwin 25.5.0, arm64
- Toolchain: rustc 1.93.1, LLVM 21.1.8
- Command: `cargo bench -p systems-snackpack-topic-003 --bench phase_workload`
- Input: 32 rounds, 100,000 iterations per phase, fixed seed, and 6.4 million
  total inner-loop iterations per trial.
- Method: eight warm-up runs followed by 31 trials; the benchmark reports the
  median.

| Metric | Median |
|---|---:|
| Whole-workload time | 21,916,333 ns |
| Inner-loop throughput | 292,019,655 iterations/s |

The timer surrounds [`run_alternating`](../src/lib.rs) and includes both phase
loops and calls between them. It excludes process startup, input construction,
sorting, and output. This run measures elapsed time, not PMU events or isolated
instruction latency. Darwin does not expose the Linux `perf` workflow.

Generated assembly came from:

```bash
cargo rustc -p systems-snackpack-topic-003 --release --lib -- --emit=asm
```

On this AArch64 build, `branch_phase` contains a data-dependent `tbz` plus its
loop-control branch. `arithmetic_phase` contains `madd`, `eor`, `subs`, and the
loop-control `b.ne`; it has no data-dependent branch. This observation applies
to this target and toolchain.
