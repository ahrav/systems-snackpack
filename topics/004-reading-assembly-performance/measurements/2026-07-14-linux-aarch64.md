## Process-level experiment

This loop starts nine benchmark processes:

```bash
for run in $(seq 1 9); do
  taskset -c 0 cargo bench --quiet \
    -p systems-snackpack-topic-004 --bench division_chain
done
```

Each process runs six warm-up pairs and 21 measured samples per path. Each
sample contains 10,000,000 loop-carried iterations. The program alternates path
order within each process and reports each path's median whole-kernel time.

| 2 | 34,665,019 ns | 83,707,696 ns | 2.414760 |
| 3 | 34,666,583 ns | 83,707,047 ns | 2.414632 |

The median of the nine constant-path process medians is 34,663,677 ns, or
3.466 ns per iteration. The constant-path process medians range from
34,662,570 to 34,666,583 ns. The median of the nine runtime-path process
medians is 83,706,095 ns, or 8.371 ns per iteration. The runtime-path process
medians range from 83,699,760 to 83,707,899 ns.

The ratio column recomputes each value from the raw nanosecond totals rather
than the three-decimal ratio printed by the benchmark. The median paired ratio
is `2.414760x`. If the nine paired ratios are independent, identically
distributed samples from one continuous process-run distribution, the second
and eighth ordered ratios form an exact 96.1% sign-based confidence interval
for that distribution's median:
`[2.414702, 2.414854]`. The interval quantifies sampling uncertainty for this
host and time block. It is not a range for future runs and does not cover build,
host, or fleet variation.

## Release codegen observation

Command:

```bash
cargo rustc --quiet -p systems-snackpack-topic-004 \
  --release --lib -- --emit=asm
```

The runtime loop contained:

```asm
madd    x10, x0, x8, x9
subs    x2, x2, #1
udiv    x0, x10, x1
b.ne    .LBB0_2
```

The constant loop contained:

```asm
madd    x11, x0, x8, x9
subs    x1, x1, #1
umulh   x12, x11, x10
sub     x11, x11, x12
add     x11, x12, x11, lsr #1
lsr     x0, x11, #2
b.ne    .LBB1_2
```

These sequences establish lowering for rustc 1.93.1 and this target. They do
not isolate instruction latency or prove behavior on another AArch64 core.

## Measurement boundary

The two clock reads span one non-inlined kernel call. The interval includes the
call and return, multiply-add, division or replacement sequence, dependency
chain, and loop control. Argument and result `black_box` calls, process startup,
warmup, validation, sorting, and output occur outside the interval.

The elapsed times and paired ratios are measured. The instruction sequences are
observed codegen. The constant and runtime paths differ in emitted division
lowering, but the timer includes every instruction in each loop. These data do
not assign the ratio to `udiv`, `umulh`, or any single core resource.
