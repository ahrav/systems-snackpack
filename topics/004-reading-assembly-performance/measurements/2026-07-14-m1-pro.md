## Whole-kernel measurement

Command:

```bash
cargo bench -p systems-snackpack-topic-004 --bench division_chain
```

Both paths use divisor `7`. Each sample runs 10,000,000 loop-carried
iterations. The program runs six warm-up pairs, then records 21 samples per
path. It alternates path order and derives each trial's seed deterministically.
It reports the median whole-kernel elapsed time. `black_box` appears only at
call boundaries.

| Path | Median total | Median per iteration |
|---|---:|---:|
| Constant divisor | 32,283,500 ns | 3.228 ns |
| Runtime divisor | 38,533,541 ns | 3.853 ns |

The measured runtime/constant ratio is `1.194x`.

The two clock reads span one non-inlined kernel call. The interval includes the
call and return plus multiply-add, division or replacement sequence, and loop
control. Argument and result `black_box` calls, warmup, validation, sorting,
process startup, and output occur outside the interval. The measurement does
not isolate divider or `umulh` latency. It does not establish performance for
another CPU.

## Local codegen observation

Command:

```bash
cargo rustc -p systems-snackpack-topic-004 --release --lib -- --emit=asm
```

The AArch64 release assembly on this host and toolchain had these loop shapes:

- Runtime divisor: `madd`, `udiv`, `subs`, `b.ne`.
- Constant divisor: `madd`, `umulh`, `sub`, corrected `add ... lsr #1`, `lsr`,
  `subs`, `b.ne`.

These lists record compiler output. They are not timing evidence.
