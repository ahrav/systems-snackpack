# Round 1: rank metadata versus working-set size

- Session: 9
- Date: 2026-07-19
- Finding: Static construction can exchange update support for compact layout,
  but succinctness is defined against a representation lower bound and an
  operation contract.
- Artifact: A two-level `rank1` directory and a full prefix oracle answer the
  same half-open query over the same deterministic bitvector.
- Space boundary: The two-level directory uses 1.3125 bits per input bit for
  large aligned inputs. Its fixed metadata ratio makes it compact, not
  formally succinct.
- Evidence boundary: Fresh processes own replication. The query timer excludes
  dataset, index, query construction, and warmup. Linked-binary disassembly
  records target-specific code generation without claiming its cause.
- Measured result: The directory used 1/24.381 of the oracle's bytes. It lost
  to the prefix oracle on the Arm host but won on `xlg`. Both hosts showed a
  timing-order effect.
- Outcome: Compare total stored semantics and measured query behavior before
  selecting a static representation. Space alone did not predict query time.
