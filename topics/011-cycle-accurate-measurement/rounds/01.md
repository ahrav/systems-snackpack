# Round 1: define the measured clock

- Session: 11
- Date: 2026-07-21
- Finding: Reference ticks, PMU cycles, and elapsed nanoseconds require separate
  names, ordering contracts, and conversion evidence.
- Artifact: The crate tests perf fixed-point endpoint conversion, a two-endpoint
  quantization guard, and one dependent recurrence. The Linux benchmark compares
  an ordered architectural counter with `CLOCK_MONOTONIC_RAW`.
- Scale boundary: x86-64 uses CPUID leaf `0x15` when its ratio and crystal
  frequency are usable. Otherwise, it takes the median of ten 100 ms
  counter-versus-clock calibrations, five in each nesting order.
- Cost boundary: Four batch sizes test whether per-operation estimates converge
  as fixed bracket cost is amortized. Every 4096-operation process median must
  reach 200 times the timer's minimum positive probe delta. This is an empirical
  granularity guard, not a timestamp-accuracy bound. The batch still includes
  recurrence loop effects and does not isolate one instruction.
- Evidence boundary: Twelve fresh processes own replication. Six processes use
  each timer order in three ABBA blocks. Every process retains 500 raw samples
  per timer and batch. Doubled integer medians preserve half units; the
  summarizer validates their derived fields and identical process checksums.
- Provenance boundary: The runner verifies a mandatory gzip-compressed Git
  archive's SHA-256 and embedded commit, then builds only its temporary
  extraction. It clears inherited Cargo-encoded Rust flags and removes the
  extraction on exit.
- Code boundary: Separate symbol-bounded linked-image gates require the selected
  three-instruction counter bracket inside `read_counter` and a multiplication
  inside `dependent_chain`. They do not assign causality to a timing ratio.
- Recorded result: Candidate `4b00356` reported batch-4096 reference-derived and
  raw-clock per-operation medians of `3.085239955` and `3.084960938 ns/op` on
  the Arm host. `xlg` reported `3.522954993` and `3.518066406 ns/op`. These are
  medians across 12 process medians, not 6000 independent observations.
- Outcome: Name the quantity and boundary before choosing a timer. Increase the
  batch until fixed harness cost fits the stated budget. Claim a quantization
  error bound only when `q` is a defensible upper bound on endpoint error.
