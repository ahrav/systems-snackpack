# Round 01: transfer locality is not placement locality

## Question

When do cache-aware tiles and cache-oblivious recursion stop predicting real
performance because finite associativity and address mapping dominate?

## Finding

Both techniques can reduce line transfers. A power-of-two leading dimension can
still concentrate active lines in a small set cycle. Padding by one element can
change that period without changing logical work. The effect must be tested on
each host because physical indexing, hashing, replacement, prefetching, and
translation are implementation details outside the ideal-cache model.

## Artifact

The crate contains naive, explicitly tiled, and recursive transpose kernels,
plus a simple modulo set-cycle calculator. The Linux runner binds measurements
to one committed archive, captures host and code-generation evidence, and uses
fresh-process order-balanced comparisons.

## Measurement boundary

The focused interval contains one transpose kernel. Allocation, source
initialization, and a 128 MiB cache-conditioning sweep share the recorded
`setup_ns` interval. Process startup and output verification are also outside
the kernel interval. A process is one replicate. Twelve blocks provide 12
paired ratios for each declared comparison.

## Interpretation boundary

Elapsed times are measured. Linked instructions are observed in the retained
disassembly. Medians, interquartile ranges, sample standard deviations, and
paired ratios are derived. Set conflict is an inference supported by a stride
perturbation and the simple modulo model, not a PMU-classified event. Results do
not generalize beyond the recorded hosts.

## Provenance boundary

The runner verifies the archive SHA-256 and its embedded Git commit before
building. It records source hashes before and after the run. This binds results
to bytes but does not independently establish trust in the Git object database.

## Generated-code boundary

The record retains the release binary checksum, symbols, focused disassembly,
and compressed full disassembly. Source-level algorithm names alone are not
evidence that the compiler preserved distinct memory-access patterns.

## Outcome

Use transfer bounds to choose an algorithm family. Use cache geometry, layout
perturbations, generated code, and paired host measurements to decide whether a
specific layout performs acceptably for the recorded host, binary, and input.
Revalidate after those conditions change.
